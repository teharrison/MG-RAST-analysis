Analysis in R: the matR extension

x

matR integrates KBase with the R language, the de facto open source standard for statistical computation.

matR can be used within R, or through the KBase iPython notebooks, as this example will illustrate.

Below we demonstrate some of the capabilities that matR offers with example analyses that were performed

as a part of Prof. Hofmockel's research.


We'll explore a data set of 120 soil samples with respect to 3 biological factors: sampling date, agricultural

treatment, and soil fraction.

This command has two sections. In the first, the user is creating a data object called "my_data" that will contain

data downloaded from KBase. This particular data request will create an object that contains two types of data:

raw abundance counts, and abundance counts with singleton counts removed that have undergone a normalization

and standardization procedure. The selected data are downloaded by matR through the KBase API.


In the second step, the downloaded data ("my_data") are saved into an R workbook ("my_dataRda").

Typically, data download and analysis would occur in the same notebook session. For the purposes of this

demonstration, we downloaded abundance data for 120 metagenomes (used in Prof. Hofmockel's analyses) and

saved them into an R workbook ("KBase-matR-IPy.Rda") that we load like this:

The data exhibit non-normal distributions, each of which are unique. This will complicate comparative analyses, making it difficult to

determine if differences observed among samples are due to biologically relevent factors, or to bias introduced by experimental and/or

sequencing error.


To account for this problem, we'll use the normalized values. We downloaded normlized values along with the raw count data. We can

look at them the same way that we looked at the raw count data. Normalized values for each sample were calculated as follows:

All values were log transformed: x_trans = log2(x + 1)

The average for transformed values in each sample was determined (x_mean) as well as the standard deviation (x_sd)

For all values in a sample i=1..n, normalized (i.e. standardized) values were calculated using the follwing formula:

x(i)_norm = ( ( x(i) - x_mean )/ x_sd )

This procedure corrects for differences in scale and location among the samples, making errant observations

due to experimetnal/seqeuencing error less likely.

After this procedure, the final nomralized values across all samples are scaled from 0 to 1.

Using Metadata to explore and analyze the data:

x

Now that we've processed the data to reduce the effect of experimental error, we can perform analyses to determine if there are trends that exhibit correlation with respect to the three factors under consideration:

  • treatment: CC=continuous, P=unfertilized prairie, PF=fertilized prairie
  • fraction: WS=whole soil, LM=large macroaggregate, MM = medium macroaggregate, SM=small macroaggregate, Micro=microaggregate
  • date: October or July of 2012

Below, R/matR is used to group data by these metadata, and we print a summary of these metadata for each sample:
